# MSW Датчик - Мониторинг

Графическое приложение для опроса датчика по Modbus RTU и отображения его параметров в реальном времени.

## Возможности

- 📊 Мониторинг параметров в реальном времени:
  - Уровень шума (дБ)
  - Температура (°C)
  - Влажность (%RH)
  - Концентрация CO₂ (ppm)
  - Освещенность (лк)
  - Качество воздуха (ppb)
  - Движение
  - Время работы устройства

- 📈 Графики в реальном времени:
  - График уровня шума (линейный с заливкой)
  - График датчика движения (ступенчатый)
  - Буфер последних 100 точек данных

- 🏛️ Управление выходами (кнопки с фиксацией):
  - Пищалка
  - Красный светодиод
  - Зеленый светодиод

- ⚙️ Настройки подключения:
  - Выбор COM-порта
  - Адрес устройства Modbus (1-247)
  - Скорость передачи (9600-115200)
  - Интервал опроса (0.1-60 сек)

## Архитектура

```
MSW-poll/
├── main.py                     # Точка входа
├── config.py                   # Конфигурация
├── requirements.txt            # Зависимости
├── ui/                         # GUI модули

│   └── main_window.py         # Главное окно
├── modbus/                     # Modbus модули
│   ├── __init__.py
│   ├── modbus_client.py       # Клиент Modbus RTU
│   └── modbus_worker.py       # Рабочий поток (QThread)
└── utils/                      # Утилиты
    ├── __init__.py
    └── logger.py              # Логирование
```

### Особенности архитектуры

- **Многопоточность**: GUI работает в главном потоке, опрос Modbus - в отдельном QThread
- **Потокобезопасность**: обмен данными через Qt signals/slots
- **Обработка ошибок**: автоматическое отключение при потере связи (3 последовательные ошибки)
- **Модульность**: разделение логики на независимые модули
- **Оптимизация**: быстрый таймаут (300мс) и блочное чтение регистров
- **Визуализация**: графики matplotlib для отображения динамики шума и движения

## Установка

1. Установите Python 3.8 или выше

2. Установите зависимости:
```bash
pip install -r requirements.txt
```

## Запуск

```bash
python main.py
```

## Использование

1. **Подключение к датчику**:
   - Выберите COM-порт из списка или введите вручную
   - Укажите адрес устройства Modbus
   - Выберите скорость передачи
   - Установите интервал опроса
   - Нажмите "Подключить"

2. **Просмотр данных**:
   - Данные автоматически обновляются с заданным интервалом
   - При ошибке датчика отображается "ошибка"

3. **Управление выходами**:
   - После подключения активируются кнопки управления с фиксацией
   - Нажимайте на кнопку для изменения состояния пищалки и светодиодов
   - Используйте "Обновить статусы" для чтения текущего состояния
   - Кнопки отображают состояние (нажата/отжата)

4. **Отключение**:
   - Нажмите "Отключить" или закройте окно
   - Настройки автоматически сохраняются в файл `settings.ini`

## Конфигурация

Настройки приложения сохраняются в файл `settings.ini` в директории приложения:

```ini
[General]
port=COM13
address=226
baudrate=9600
parity=N
poll_interval=0.1
```

Файл можно редактировать вручную при закрытом приложении.

## Протокол Modbus

### Holding-регистры (чтение, function code 3):
- 3: Уровень шума (×0.01, дБ)
- 4: Температура (×0.01, °C, 0x7FFF = ошибка)
- 5: Влажность (×0.01, %RH, 0xFFFF = ошибка)
- 8: CO₂ (ppm, 0xFFFF = ошибка)
- 9-10: Освещенность (u32, ×0.01, лк, 9 = старшее слово)
- 11: Качество воздуха (ppb, 0xFFFF = ошибка)
- 104-105: Время работы (u32, секунды)
- 283: Движение (0xFFFF = ошибка)

**Оптимизация:** Регистры читаются блоками (3-5, 8-11, 104-105, 283), что ускоряет опрос.

### Coil-регистры (чтение/запись, function codes 1/5):
- 0: Пищалка
- 10: Красный светодиод
- 11: Зеленый светодиод

**Особенности:** Coils читаются и записываются последовательно, так как устройство не поддерживает групповое чтение/запись.

## Устранение неполадок

### "Slave reported illegal data address"
- Проверьте правильность адреса устройства (обычно 1-247)
- Убедитесь, что регистры существуют в вашем устройстве
- Проверьте документацию устройства для точных адресов регистров

### "No communication with the instrument"
- Проверьте правильность COM-порта
- Убедитесь, что скорость передачи соответствует настройкам устройства
- Попробуйте изменить настройку четности (None/Even/Odd)
- Проверьте физическое подключение кабеля
- При подключении к пустому порту фиксируется отсутствие устройства за ~1 секунду

### Потеря связи
При потере связи (3 последовательные ошибки):
- Кнопка автоматически меняется на "Подключить"
- Элементы управления отключаются
- Настройки подключения разблокируются
